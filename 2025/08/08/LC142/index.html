LC142 - 环形链表 II

原题

142. 环形链表 II - 力扣（LeetCode）

给定一个链表的头节点 head ，返回链表开始入环的第一个节点。 如果链表无环，则返回 null。

如果链表中有某个节点，可以通过连续跟踪 next 指针再次到达，则链表中存在环。 为了表示给定链表中的环，评测系统内部使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。如果 pos 是 -1，则在该链表中没有环。注意：pos 不作为参数进行传递，仅仅是为了标识链表的实际情况。

题解

哈希表

时间复杂度：O(n)，其中 n 为链表中节点的数目。我们恰好需要访问链表中的每一个节点。

空间复杂度：O(n)，其中 n 为链表中节点的数目。我们需要将链表中的每个节点都保存在哈希表当中

快慢指针

时间复杂度：O(n)，其中 n 为链表中节点的数目。在最初判断快慢指针是否相遇时，slow 指针走过的距离不会超过链表的总长度；随后寻找入环点时，走过的距离也不会超过链表的总长度。因此，总的执行时间为 O(n) + O(n) = O(n)。

空间复杂度：O(1)。我们只使用了 slow,quick 两个指针。

总结

分析一下这个快慢指针的方法，先看一张图：

a为环前的节点数量，b为入环的第一个节点到相遇点的节点数量，c为相遇点到入环第一个节点的节点数量。

先看第一个过程，快指针追上慢指针：

slow = a + b（快指针一定在慢指针走完一圈前追上，快指针比慢指针速度快1，而他们之间的距离最长为b + c − 1，也就是环长-1。）

quick = a + b + n(b + c)（快指针在追上慢指针时，比其多走几个圈长）

quick = 2 * slow（快指针速度是慢指针的两倍，所以走的距离也是慢指针的两倍）

带入quick和slow，2 * (a + b) = a + b + n(b + c)，而题目要求的相当于就是a，移项得到a = n * (b + c) − b 等价于 a = n * (b + c) + c。

得到的式子代表a的长度相当于c的长度加上若干圈长，此时一个指针从head出发，一个指针从相遇点出发，两个指针速度相同，那么二者一定会在入环处碰头，最后两个指针相遇时，相遇点指向的就是入环处。
